# Training Plan — Analyze Phase

## English
This folder contains a case-agnostic, step-by-step training guide for the DMAIC **Analyze** phase. It helps teams transition from identifying symptoms to proving the true root causes of a problem using data and logic.

### Step-by-Step Training Guide
1. **Brainstorm Potential Causes:** Use an Ishikawa (Fishbone) Diagram to capture all possible reasons for the defect.
2. **Filter Potential Causes:** Use a Cause & Effect Matrix to prioritize the most likely causes based on expert knowledge.
3. **Drill Down to Root Causes:** Apply the "5 Whys" technique to the top prioritized causes.
4. **Collect Data for Verification:** Gather data specifically targeting the suspected root causes.
5. **Verify Root Causes Statistically:** Use Hypothesis Testing (e.g., ANOVA, T-Test, Chi-Square) or Graphical Analysis (e.g., Scatter Plots, Box Plots) to prove the root cause.

## Arabic
# خطة التدريب — مرحلة التحليل

يحتوي هذا المجلد على دليل تدريبي تدريجي عام يصلح لأي حالة دراسية لمرحلة **التحليل** في منهجية 6S. يساعد الفرق على الانتقال من تحديد الأعراض إلى إثبات الأسباب الجذرية الحقيقية للمشكلة باستخدام البيانات والمنطق.

### دليل التدريب خطوة بخطوة
1. **العصف الذهني للأسباب المحتملة:** استخدم مخطط إيشيكاوا (عظم السمكة) لالتقاط جميع الأسباب المحتملة للعيب.
2. **تصفية الأسباب المحتملة:** استخدم مصفوفة السبب والأثر لترتيب الأسباب الأكثر احتمالاً بناءً على معرفة الخبراء.
3. **التعمق إلى الأسباب الجذرية:** طبّق تقنية "لماذا الخمس" على الأسباب ذات الأولوية القصوى.
4. **جمع البيانات للتحقق:** اجمع البيانات التي تستهدف تحديداً الأسباب الجذرية المشتبه بها.
5. **التحقق من الأسباب الجذرية إحصائياً:** استخدم اختبار الفرضيات (مثل تحليل التباين، اختبار T، مربع كاي) أو التحليل الرسومي (مثل المخططات المبعثرة، المخططات الصندوقية) لإثبات السبب الجذري.
