# D2_Measure Phase Guide (Template)

## English
The **Measure** phase focuses on understanding the current state of the process and establishing a reliable baseline. This involves deciding what to measure, ensuring the measurement system is accurate, and collecting data to determine current process capability.

### Step-by-Step Checklist
- [ ] Develop a detailed Data Collection Plan (What, How, Who, When).
- [ ] Validate the measurement system using Measurement System Analysis (MSA) / Gage R&R.
- [ ] Collect baseline data (historical or real-time) according to the plan.
- [ ] Calculate current Process Capability (Sigma Level, DPMO, Yield, Cp, Cpk).
- [ ] Map the detailed current state process (e.g., Value Stream Mapping).
- [ ] Communicate data collection requirements to the relevant teams.

### Best Practices
- **Trust the Data, Not Opinions:** Ensure decisions are based on validated data, not anecdotal evidence.
- **Validate the Measurement System:** If your measurement system is flawed, your baseline and subsequent improvements will be invalid. Always perform an MSA.
- **Understand Variation:** Look for both common cause and special cause variation in your baseline data.

## Arabic
# دليل مرحلة D2_Measure (قالب)

تركز مرحلة **القياس** على فهم الحالة الحالية للعملية وإنشاء خط أساس موثوق. يتضمن ذلك تحديد ما يجب قياسه، وضمان دقة نظام القياس، وجمع البيانات لتحديد قدرة العملية الحالية.

### قائمة التحقق خطوة بخطوة
- [ ] تطوير خطة مفصلة لجمع البيانات (ماذا، كيف، من، متى).
- [ ] التحقق من صحة نظام القياس باستخدام تحليل نظام القياس (MSA) / دراسة التكرارية والتوافقية (Gage R&R).
- [ ] جمع بيانات خط الأساس (تاريخية أو في الوقت الفعلي) وفقًا للخطة.
- [ ] حساب قدرة العملية الحالية (مستوى 6S، العيوب لكل مليون فرصة، العائد، Cp، Cpk).
- [ ] رسم خريطة مفصلة للعملية الحالية (مثل خريطة تدفق القيمة).
- [ ] توصيل متطلبات جمع البيانات للفرق المعنية.

### أفضل الممارسات
- **ثق بالبيانات، وليس الآراء:** تأكد من أن القرارات تستند إلى بيانات تم التحقق من صحتها، وليس إلى أدلة قصصية.
- **التحقق من نظام القياس:** إذا كان نظام القياس الخاص بك معيبًا، فسيكون خط الأساس والتحسينات اللاحقة غير صالحة. قم دائمًا بإجراء تحليل نظام القياس (MSA).
- **فهم التباين:** ابحث عن كل من التباين ذي السبب الشائع والتباين ذي السبب الخاص في بيانات خط الأساس الخاصة بك.
