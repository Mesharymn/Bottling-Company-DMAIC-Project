# D6_Guide — DMAIC Case-Agnostic Template

## English

This directory is a **complete, reusable DMAIC template** designed to guide any project team through all five phases of the 6S (Lean Six Sigma) methodology. It is fully case-agnostic and can be applied to any industry or process type, including IT, manufacturing, logistics, healthcare, and services.

Each phase folder mirrors the structure of the active project phases (D1–D5) and contains step-by-step guides, checklists, best practices, communication templates, and links to real external resources.

### How to Use This Guide
1. Start at `D1_Define/` and work through each phase in order.
2. Open the `README.md` in each phase folder for a step-by-step guide and checklist.
3. Use the subfolders to store your project's deliverables for that phase.
4. Use the `05_Communications/` subfolder in each phase for all emails and meeting minutes.

### Structure

| Phase | Folder | Key Subfolders |
|-------|--------|----------------|
| **Define** | `D1_Define/` | Project Charter, Process Mapping, Voice of Customer, Stakeholder Analysis, Communications |
| **Measure** | `D2_Measure/` | Data Collection Plan, Measurement System Analysis, Process Capability, Value Stream Mapping, Communications |
| **Analyze** | `D3_Analyze/` | Fishbone Diagram, 5 Why Analysis, Hypothesis Testing, Lean Waste Analysis, Root Cause Summary, Communications |
| **Improve** | `D4_Improve/` | Solutions Selection, Implementation Plan, Pilot Testing, Change Management, Training Plan, Communications |
| **Control** | `D5_Control/` | Standard Operating Procedures, Control Plan, Statistical Process Control, Training Handover, Communications |

### Key Principles
- **Design for Humans:** The best solution is one that makes the right behavior the easiest behavior.
- **Validate Everything:** Every root cause and every solution must be validated with data, not assumptions.
- **Sustain the Gains:** A project is not finished when the solution is implemented. It is finished when the Control phase is complete and the Process Owner has formally accepted the handover.

## Arabic
# D6_Guide — قالب 6S العام (DMAIC)

هذا الدليل هو **قالب 6S كامل وقابل لإعادة الاستخدام** مصمم لإرشاد أي فريق مشروع خلال جميع المراحل الخمس لمنهجية 6S (الإدارة الرشيقة وستة سيغما). إنه عام تمامًا ويمكن تطبيقه على أي صناعة أو نوع عملية، بما في ذلك تقنية المعلومات والتصنيع واللوجستكس والرعاية الصحية والخدمات.

تعكس كل مجلد مرحلة هيكل مراحل المشروع النشطة (D1–D5) وتحتوي على أدلة خطوة بخطوة وقوائم تحقق وأفضل الممارسات وقوالب التواصل وروابط للموارد الخارجية الحقيقية.

### كيفية استخدام هذا الدليل
1. ابدأ من `D1_Define/` واعمل خلال كل مرحلة بالترتيب.
2. افتح `README.md` في كل مجلد مرحلة للحصول على دليل خطوة بخطوة وقائمة تحقق.
3. استخدم المجلدات الفرعية لتخزين مخرجات مشروعك لتلك المرحلة.
4. استخدم المجلد الفرعي `05_Communications/` في كل مرحلة لجميع رسائل البريد الإلكتروني ومحاضر الاجتماعات.

### الهيكل

| المرحلة | المجلد | المجلدات الفرعية الرئيسية |
|---------|--------|---------------------------|
| **التعريف** | `D1_Define/` | ميثاق المشروع، رسم خريطة العملية، صوت العميل، تحليل أصحاب المصلحة، التواصل |
| **القياس** | `D2_Measure/` | خطة جمع البيانات، تحليل نظام القياس، قدرة العملية، خريطة تدفق القيمة، التواصل |
| **التحليل** | `D3_Analyze/` | مخطط عظم السمكة، تحليل لماذا الخمس، اختبار الفرضيات، تحليل هدر الإدارة الرشيقة، ملخص الأسباب الجذرية، التواصل |
| **التحسين** | `D4_Improve/` | اختيار الحلول، خطة التنفيذ، الاختبار التجريبي، إدارة التغيير، خطة التدريب، التواصل |
| **التحكم** | `D5_Control/` | إجراءات التشغيل القياسية، خطة التحكم، التحكم الإحصائي في العمليات، تسليم التدريب، التواصل |

### المبادئ الرئيسية
- **التصميم للبشر:** أفضل حل هو الذي يجعل السلوك الصحيح هو السلوك الأسهل.
- **التحقق من كل شيء:** يجب التحقق من كل سبب جذري وكل حل بالبيانات، وليس بالافتراضات.
- **الحفاظ على المكاسب:** المشروع لا ينتهي عند تنفيذ الحل. ينتهي عند اكتمال مرحلة التحكم وقبول مالك العملية رسميًا للتسليم.
