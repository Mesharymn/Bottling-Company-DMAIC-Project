# D4 — Improve Phase
## Water Bottling Company | DMAIC Project

> The Improve phase answers one question: **"How do we fix the root causes and verify the solutions?"**

---

| Subfolder | Tool/Artifact | Purpose |
|-----------|---------------|---------|
| `01_Solutions_Selection/` | Impact/Effort & Pugh Matrix | Evaluate and select the best solutions for confirmed root causes |

| `03_Pilot_Testing/` | Pilot Plan & Results | Test solutions on a small scale before full deployment |

---

## Improvement Plan for Root Causes

Based on the Analyze phase findings, we are deploying the following solutions to address the confirmed root causes:

| Root Cause | Proposed Solution | Expected Impact | ROI Strategy |
|------------|-------------------|-----------------|--------------|
| **RC-01:** Outdated Maintenance Plan | Update PM plan + implement TPM | 60% reduction in leakage | Calculate downtime savings vs TPM cost |
| **RC-02:** No Formal PM Program | Deploy formal PM schedule | 40% reduction in downtime | Monitor MTBF improvement |
| **RC-03:** No SMED Procedure | Implement SMED for all SKU changes | Changeover time ≤ 20 mins | Value of regained production capacity |
| **RC-04:** Worn Torque Sensor | Replace + calibrate torque sensor | Eliminate cap deformation | Cost of scrap reduction |
| **RC-05:** No CI Culture | Launch Kaizen program + Suggestion system | Sustain improvements | Track implemented suggestions |
| **RC-06:** Push Production | Implement Kanban pull system | Eliminate overproduction waste | Reduce WIP inventory costs |

---

## Behavior Design & Change Management (ADKAR & COM-B)

To ensure the technical solutions succeed, we must design for the human element:
- **Capability:** Train operators on the new SMED and PM procedures.
- **Opportunity:** Provide the right tools and visual cues (Poka-Yoke) to make the new process the default.
- **Motivation:** Use ADKAR to build awareness and desire for the change before training begins.

## خطة التحسين للأسباب الجذرية

بناءً على نتائج مرحلة التحليل، سنقوم بتطبيق الحلول التالية لمعالجة الأسباب الجذرية المؤكدة:

| السبب الجذري | الحل المقترح | التأثير المتوقع | استراتيجية العائد على الاستثمار (ROI) |
|-------------|------------|----------------|-------------------------------------|
| **RC-01:** خطة الصيانة قديمة | تحديث خطة الصيانة الوقائية + تنفيذ TPM | تخفيض التسرب بنسبة 60% | حساب توفير التوقف مقابل تكلفة TPM |
| **RC-02:** لا يوجد برنامج PM | تطبيق جدول صيانة وقائية رسمي | تخفيض التوقف بنسبة 40% | مراقبة تحسن متوسط الوقت بين الأعطال (MTBF) |
| **RC-03:** لا يوجد إجراء SMED | تنفيذ SMED لجميع تحويلات SKU | وقت التحويل ≤ 20 دقيقة | قيمة الطاقة الإنتاجية المستردة |
| **RC-04:** مستشعر عزم متآكل | استبدال ومعايرة مستشعر العزم | القضاء على تشوه الغطاء | تقليل تكلفة الخردة (Scrap) |
| **RC-05:** لا توجد ثقافة CI | إطلاق برنامج Kaizen ونظام اقتراحات | الحفاظ على التحسينات | تتبع الاقتراحات المنفذة وقيمتها |
| **RC-06:** إنتاج بالدفع (Push) | تنفيذ نظام سحب Kanban | القضاء على هدر الإنتاج الزائد | تقليل تكاليف المخزون قيد التشغيل (WIP) |

---

## تصميم السلوك وإدارة التغيير (ADKAR و COM-B)

لضمان نجاح الحلول التقنية، يجب أن نصمم للعنصر البشري:
- **القدرة (Capability):** تدريب المشغلين على إجراءات SMED والصيانة الوقائية الجديدة.
- **الفرصة (Opportunity):** توفير الأدوات المناسبة والإشارات البصرية (Poka-Yoke) لجعل العملية الجديدة هي الخيار الافتراضي.
- **الدافع (Motivation):** استخدام نموذج ADKAR لبناء الوعي والرغبة في التغيير قبل بدء التدريب.
